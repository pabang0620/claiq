# CLAIQ 심사자 데모 가이드

> **소요 시간: 약 10분** | 별도 설치 없음 | 계정 생성 불필요

---

## 빠른 시작

**라이브 URL**: [https://claiq.vercel.app](https://claiq.vercel.app)

로그인 화면 하단의 **"데모 계정으로 빠른 시작"** 버튼을 클릭하면 즉시 입장됩니다.
비밀번호 입력 없이 한 번의 클릭으로 각 역할로 전환할 수 있습니다.

---

## 권장 탐색 순서 (10분)

### 1단계 — 운영자 시점 (3분)
> 버튼: **운영자 · 정민석**

1. **대시보드** 진입
   - 이탈 위험 수강생 수, 출석률, 미발송 리포트 등 핵심 지표 한눈에 확인
   - `at_risk`(3일 미활동) / `inactive`(7일 미활동) 분류 현황

2. **이탈 예측** 메뉴 클릭
   - 수강생별 위험도 분류 + AI 코멘트 확인
   - 출결 데이터 기반 자동 감지 (rule-based churn detection)

3. **성취 리포트** 메뉴 클릭
   - 수강생별 성취도 자동 생성 리포트 미리보기
   - 집중학습 필요 유형(약점)이 한국어 유형명으로 표시됨
   - SMS 발송 버튼으로 학부모 통보 시뮬레이션

---

### 2단계 — 교강사 시점 (4분)
> 버튼: **교강사 · 이준혁**

1. **강의 목록** 진입
   - 기존 강의에 문제가 이미 생성된 상태 확인

2. **강의 하나 클릭** → "AI 문제 생성" 버튼
   - Whisper STT → GPT-4o-mini 문제 자동생성 파이프라인 실행
   - 우측에 SSE 실시간 진행상황 바 표시
   - **실제 OpenAI API가 호출됩니다** (5지선다 수능형 문제 자동 생성)

3. **문제 검수** 메뉴 클릭
   - `pending` 탭: AI가 생성한 문제 검토
   - 승인(→ 수강생 출제), 편집, 반려 가능

4. **QA 에스컬레이션** 메뉴 클릭
   - AI가 처리하지 못한 질문 확인 및 직접 답변

---

### 3단계 — 수강생 시점 (3분)
> 버튼: **수강생 · 김민준** (또는 다른 수강생 선택)

1. **대시보드**
   - 수능 D-day 카운트다운, 오늘의 스트릭, 포인트, 로드맵 진행률 확인

2. **오늘의 문제** 메뉴
   - 교강사가 승인한 수능형 문제 출제
   - 5지선다 즉시 채점 + 상세 해설 제공

3. **AI Q&A** 메뉴
   - 강의 내용 기반 질문 입력 (예: "미적분 극값의 조건을 다시 설명해줘")
   - pgvector 코사인 유사도 검색 → GPT-4o-mini RAG 스트리밍 답변
   - AI가 답하지 못하면 교강사에게 에스컬레이션

4. **약점 분석** 메뉴
   - 수능 유형별 정답률 차트
   - 집중학습 필요 Top 5 유형 목록

5. **D-day 로드맵** 메뉴
   - 수능까지 남은 일수 역산 → 주간 학습 계획 자동 배분
   - 항목 완료 체크 가능

## 핵심 기술 포인트 (심사 체크포인트)

| 항목 | 구현 내용 |
|------|----------|
| **AI 파이프라인 복잡도** | Whisper STT → 임베딩 → pgvector 저장 → 수능 유형 매핑 → 문제 생성 (5단계 비동기 파이프라인) |
| **실시간 통신** | SSE(Server-Sent Events)로 AI 처리 진행상황 및 Q&A 답변 스트리밍 |
| **RAG 구현** | text-embedding-3-small + pgvector 코사인 유사도 검색, 교강사별 강의 범위 필터 |
| **Human-in-the-Loop** | AI 생성 문제를 교강사가 반드시 검수·승인해야 수강생 출제 활성화 |
| **D-day 역산 로드맵** | 수능 날짜 고정 → 남은 일수 기반 유형별 우선순위 역산 알고리즘 |
| **이탈 예측** | 출결 기반 at_risk/inactive 자동 분류, 매일 오전 9시 크론 감지 |
| **3-role 생태계** | 운영자·교강사·수강생이 같은 학원 안에서 데이터 공유 |
| **실 배포** | Vercel(프론트) + Render(백엔드) + Supabase(PostgreSQL+pgvector) |

## 데모 데이터 구성

| 항목 | 내용 |
|------|------|
| 학원 | 대치 클레이크 학원 |
| 강의 | 5개 (국어독서, 문학, 수학미적분, 영어독해) |
| 수능형 문제 | 7개 (5지선다 + 단답형, 수능 실제 유형 기반) |
| 수강생 | 5명 (정답률 86%~29% 현실적 분포) |
| 출석 기록 | 3주치 |
| Q&A 세션 | 수강생별 2~3개 |
| 성취 리포트 | 수강생 전원 생성 완료 |

## AI 협업 개발 방식

CLAIQ는 Claude Code와 10개 전문 에이전트를 병렬 활용해 **2인 팀**이 단기간에 구현했습니다.

```
architect 에이전트    → DB 27테이블 설계, RAG 파이프라인 아키텍처
express-engineer 에이전트 → 12개 도메인 API, AI 모듈 7개
react-specialist 에이전트 → 32개 페이지, 9개 Zustand 스토어
database-reviewer 에이전트 → 마이그레이션 검증, 쿼리 최적화
security-reviewer 에이전트 → JWT 인증, SQL 인젝션 방지, 환경변수 관리
```
