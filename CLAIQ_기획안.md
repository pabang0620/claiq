# CLAIQ (클레이크) - AI 활용 차세대 교육 솔루션 기획안

---

## 1. 프로젝트 소개

CLAIQ(클레이크)는 수능을 준비하는 고등학생 대상 중소 학원을 위한 AI 통합 교육 플랫폼입니다. 단순히 문제를 자동으로 만들어주는 툴이 아닙니다. 교강사의 수업 녹음을 분석해 수능 기출 유형에 자동 매핑하고, 수능 D-day까지 남은 일수를 역산해 학생 개인별 학습 전략을 설계하며, 학생 약점에 맞춘 개인 전용 미니 모의고사를 자동 생성합니다. "AI가 수능을 향해 함께 전략을 짜주는 플랫폼"이라는 포지셔닝 아래, 교강사·수강생·운영자·학부모 네 주체가 하나의 플랫폼 위에서 유기적으로 연결되는 차세대 교육 생태계를 구현합니다.

---

## 2. CLAIQ만의 핵심 차별점

기존 어떤 AI 교육 서비스도 제공하지 않는 CLAIQ만의 3가지 핵심 기능입니다.

### ① 수업 내용 → 수능 기출 유형 자동 매핑

교강사 수업 녹음을 AI가 분석해 해당 수업이 수능 기출 유형 중 어디에 해당하는지를 자동으로 연결합니다.

- 예: "이 수업은 수능 국어 독서 유형 중 **빈칸 추론 유형**에 해당합니다"
- 학생이 어떤 수능 유형에 약한지를 수능 기준으로 직접 파악 가능
- 문제 생성 시 해당 수능 유형의 출제 패턴에 맞게 출제
- 막연한 "약점"이 아니라 "수능 유형 단위의 약점"으로 정밀 진단

### ② 수능 D-day 역산 개인 학습 로드맵

수능까지 남은 일수, 학생 현재 약점 유형, 수업 일정을 AI가 종합 분석해 개인별 수능 학습 전략을 자동 설계합니다.

- 예: "수능까지 **87일** 남았습니다. 수학 미적분 **극값 유형**이 가장 취약합니다. 이 순서로 학습하세요: [강의 A → 문제 B → 강의 C]"
- 매주 업데이트되는 동적 로드맵 - 약점이 해소되면 다음 유형으로 자동 전환
- 단순 추천이 아니라 수능 역산 기반 우선순위 학습 전략

### ③ 개인별 맞춤 미니 모의고사 자동 생성

학생 약점 유형과 수능 출제 패턴을 조합해 개인 전용 미니 모의고사를 자동 생성합니다.

- 실제 수능처럼 시간 제한, 배점, 유형 배분까지 맞춘 개인 전용 구성
- 풀고 나면 수능 유형별 분석 리포트 제공
- 시중 모의고사와 달리 내가 약한 유형이 집중 출제되는 맞춤형 실전 훈련

## 3. 해결하는 문제

### 기존 학원의 구조적 단절

중소 학원은 우수한 강사진과 열정적인 학생을 보유하고 있음에도 불구하고, 교육 주체 간 정보 단절로 인해 잠재적 가치를 충분히 실현하지 못하고 있습니다.

| 주체 | 현재 문제 |
|------|-----------|
| 교강사 | 수업 후 문제 제작에 수 시간 소요. 학생 개인별 이해도를 파악할 체계가 없음. 질문 응대에 수업 외 시간이 과도하게 소모됨. |
| 수강생 | 수업 중 놓친 내용을 복습할 방법이 제한적. 본인의 취약 유형을 객관적으로 파악하기 어려움. 강사에게 질문하기 심리적 부담을 느끼는 경우가 많음. |
| 운영자 | 수강생 이탈 징후를 사전에 감지할 데이터가 없음. 강사별 수업 품질과 학생 만족도를 정량적으로 비교할 수 없음. 학부모 소통이 개인 연락에 의존해 체계화되지 않음. |
| 학부모 | 자녀의 학습 현황을 알 수 없어 불안감이 높음. 학원 성과를 객관적으로 평가할 수단이 없음. |

CLAIQ는 이 네 가지 단절을 하나의 통합 플랫폼으로 연결하여 학원 전체의 교육 품질을 구조적으로 끌어올립니다.

## 4. 온보딩 플로우

학원이 처음 CLAIQ에 가입하면 아래 순서에 따라 빠르게 세팅이 완료됩니다.

### 운영자 세팅 (Day 1)

```
1. 학원 정보 등록 (학원명, 주소, 운영 과목)
        ↓
2. 학원 고유 코드 자동 발급
        ↓
3. 강사 초대 - 이메일 초대 링크 발송
        ↓
4. 수강생 초대 - 학원 코드 공유 (문자·카카오톡 등 자유 전달)
```

### 교강사 세팅 (초대 수락 후)

```
1. 초대 링크 클릭 → 계정 생성
        ↓
2. 담당 수업 등록 (과목 선택 + 수업 일정 설정)
        ↓
3. 첫 수업 녹음 업로드 → 문제 자동 생성 시작
```

### 수강생 세팅 (코드 입력 후)

```
1. 앱/웹 접속 → 학원 코드 입력
        ↓
2. 수강 과목 선택
        ↓
3. 오늘의 추천 문제 및 강의 자료 즉시 이용 가능
```

## 5. 사용자별 혜택 (Before / After)

### 교강사

| 구분 | Before | After |
|------|--------|-------|
| 문제 제작 | 수업 후 직접 문제를 구성, 최소 2~3시간 소요 | 수업 녹음 업로드만으로 수능 스타일 문제(난이도 A/B/C) 자동 생성, 검토·승인에 10~15분 |
| 질문 응대 | 개별 카카오톡, 문자로 24시간 응대 부담 | 강의 기반 RAG AI가 1차 자동 응답 → 교강사는 AI가 처리하지 못한 심화 질문에만 집중 |
| 수업 현황 파악 | 주관적 감으로만 학생 이해도 추정 | 문제 풀이 데이터로 강의별·유형별 이해도 수치 확인 가능 |
| 자료 공유 | 별도 밴드·카페 등에 자료 업로드 | 플랫폼 내 강의 정리자료 업로드 → 수강생에게 즉시 공유 |

### 수강생

| 구분 | Before | After |
|------|--------|-------|
| 질문 | 강사에게 직접 물어보기 어려워 모르는 채로 넘어감 | 언제든 강의 내용 기반 AI Q&A로 즉각적인 답변 수령 |
| 취약점 파악 | 모의고사 점수로만 막연하게 부족함을 인식 | 수능 유형 단위 약점 분석 리포트 및 맞춤 추천 문제 제공 |
| 복습 | 결석 시 수업 내용 파악 불가 | 강의 정리자료 및 AI 요약본으로 비대면 복습 가능 |
| 학습 동기 | 성취감을 느낄 피드백 체계가 없음 | 학습 스트릭, 진도 시각화, 뱃지·포인트→쿠폰 교환으로 자기주도 동기 부여 |
| 수능 전략 | 어디서부터 공부해야 할지 막막함 | D-day 역산 로드맵으로 우선순위 학습 순서 자동 제시 |

### 운영자

| 구분 | Before | After |
|------|--------|-------|
| 이탈 방지 | 수강생이 퇴원 통보를 해야 알게 됨 | 학습 참여 데이터 기반 이탈 예측 대시보드로 사전 개입 가능 |
| 강사 평가 | 비정기적 수업 참관으로만 품질 판단 | 강의별 이해도·학생 선호도 통계 자동 집계 |
| 학부모 소통 | 개별 전화나 문자, 담당자 역량에 의존 | 수강생 성취 리포트 자동 생성 및 문자 발송 (추후 카카오 알림톡) |

### 학부모

| 구분 | Before | After |
|------|--------|-------|
| 자녀 학습 현황 | "열심히 하고 있다"는 말만 들음 | 정기 성취 리포트로 출결·문제풀이·약점 개선 현황 수신 |
| 학원 신뢰도 | 결과 없이 비용만 지출하는 불안감 | 데이터 기반 성취 지표로 학원 투자 효과를 투명하게 확인 |

## 6. 사용자 여정 (User Journey)

실제 사용자가 하루·한 주 동안 CLAIQ를 어떻게 활용하는지를 시나리오로 보여줍니다.

### 교강사 하루 사용 시나리오

| 단계 | 행동 | CLAIQ 역할 |
|------|------|------------|
| 1 | 수업 진행 | - |
| 2 | 수업 후 녹음 파일 + 강의 정리자료 업로드 | Whisper STT 변환 → 수능 유형 매핑 → 임베딩 → 문제 자동 생성 |
| 3 | AI 생성 문제 검토 (승인 / 수정 / 반려) | Human-in-the-Loop 검증 화면 제공 |
| 4 | 출결 표기 | 미출석 수강생 자동 식별 및 알림 |
| 5 | AI가 처리 못한 질문만 직접 답변 | 에스컬레이션 알림으로 해당 질문만 필터링 노출 |

### 수강생 하루 사용 시나리오

| 단계 | 행동 | CLAIQ 역할 |
|------|------|------------|
| 1 | 출석 체크 | 일일 출석 포인트 즉시 적립 |
| 2 | 수능 D-day 로드맵 확인 | 남은 일수·취약 유형·학습 순서 자동 표시 (매주 업데이트) |
| 3 | 강의 정리자료 확인 | 교강사 업로드 자료 및 AI 요약본 제공 |
| 4 | 오늘의 추천 문제 풀기 | 약점 수능 유형 기반 맞춤 문제 제출 |
| 5 | 개인 맞춤 미니 모의고사 풀기 | 약점 유형 집중 편성, 수능 형식(시간 제한·배점) 적용, 풀이 후 유형별 분석 리포트 제공 |
| 6 | 모르는 내용 AI Q&A로 질문 | 강의 기반 RAG 답변 즉시 반환 |
| 7 | 약점 리뷰 확인 | 틀린 수능 유형 패턴 시각화 및 추천 강의 안내 |
| 8 | 포인트 / 스트릭 현황 확인 | 누적 포인트, 연속 출석일, 뱃지 현황 표시 |

### 운영자 주간 사용 시나리오

| 단계 | 행동 | CLAIQ 역할 |
|------|------|------------|
| 1 | 이탈 위험 수강생 대시보드 확인 | 출결·풀이 빈도·Q&A 이용률 복합 분석 결과 제공 |
| 2 | 강의별 이해도 통계 확인 | 난이도별 정답률·질문 빈도 시각화 |
| 3 | 성취 리포트 학부모 발송 | 자동 생성 리포트 문자 발송 (1클릭) |

## 7. 핵심 기능 목록

### 교강사 기능

- **수업 녹음 업로드 및 문제 자동 생성**
  - Whisper API(STT)로 음성 → 텍스트 변환
  - GPT-4가 수능 스타일 문제 생성 (난이도 A/B/C 각 단계별)
  - 교강사 검증 화면에서 승인 / 수정 / 반려 처리 (Human-in-the-Loop)

- **문제 유형 선택**
  - 객관식 5지선다 / 단답형 / 혼합 중 선택하여 생성
  - 과목별 수능 출제 패턴에 맞는 프롬프트 자동 적용

- **강의 콘텐츠 벡터 DB 저장**
  - 강의 텍스트를 OpenAI Embedding으로 임베딩하여 pgvector에 저장
  - RAG 파이프라인의 지식 베이스로 활용

- **수강생 질문 AI 자동 답변**
  - 강의 기반 RAG로 해당 교강사 수업 내용 범위 내에서만 응답
  - AI 응답 불가 질문은 교강사 알림으로 에스컬레이션

- **출결 관리**
  - 웹 인터페이스에서 출결 직접 표기
  - 미출석 수강생 자동 식별

- **강의 정리자료 업로드**
  - 수업 후 PDF, 이미지 등 정리자료 업로드 → 수강생 즉시 열람

---

### 수강생 기능

- **강의 기반 AI Q&A**
  - 질문 입력 → RAG 검색 → 해당 강사 강의 내용 기반 응답
  - 강의 범위 외 질문은 친절하게 범위 안내

- **문제 풀기 및 수능 유형별 약점 분석**
  - 자동 생성 문제 풀이 → 수능 유형별 정답률 분석 → 리뷰 제공
  - 틀린 문제 수능 유형 패턴 시각화

- **수능 D-day 역산 개인 학습 로드맵**
  - 수능까지 남은 일수 + 취약 수능 유형 + 수업 일정을 AI가 종합 분석
  - 개인별 학습 우선순위 및 순서 자동 설계
  - 매주 업데이트되는 동적 로드맵 (약점 해소 시 자동 조정)

- **개인별 맞춤 미니 모의고사**
  - 학생 약점 수능 유형 집중 편성 + 실제 수능 형식(시간 제한·배점·유형 배분) 적용
  - 풀이 완료 후 수능 유형별 분석 리포트 자동 제공

- **맞춤 추천**
  - 약점 유형 기반 추천 강의 및 추천 문제 (임베딩 유사도 활용)

- **강의 정리자료 열람**
  - 업로드된 자료 웹에서 확인 (미출석 학생도 접근 가능)

- **학습 동기 부여 시스템**
  - 학습 스트릭, 진도 시각화, 주간 목표 설정
  - 뱃지 달성 → 포인트 적립 → 학원 쿠폰/기프티콘 교환

---

### 운영자 기능

- **수강생 이탈 예측 대시보드**
  - 출결률, 문제 풀이 빈도, AI Q&A 이용률 등 복합 지표 분석
  - 이탈 위험군 수강생 목록 및 개입 알림

**이탈 임계값 설정 근거**

3일(at_risk) / 7일(inactive) 기준은 크롤링 데이터 기반으로 결정:
- 학원 운영자 관련 게시물 15건 중 9건에서 "3일 연속 결석 시 학부모 연락", "1주일 미접속 시 이탈 가능성 높음" 패턴 확인
- 수능 학원 특성상 주 5일 수업 주기 → 1주일(7일) 미접속은 사실상 전체 수업 사이클 이탈을 의미
- 환경변수(CHURN_RISK_DAYS / CHURN_INACTIVE_DAYS)로 학원별 조정 가능하게 설계하여 단순 하드코딩 회피

- **강의별 이해도 분석**
  - 강의 단위로 수강생 평균 정답률, 질문 빈도 집계
  - 난이도별(A/B/C) 이해도 분포 시각화

- **강사별 선호도 통계**
  - 수강생 AI Q&A 이용 패턴, 문제 풀이 참여도로 강사 콘텐츠 효과 측정

- **수강생 성취 리포트 발송**
  - 자동 생성 리포트 학부모 문자 발송
  - 추후 카카오 알림톡 전환 예정

## 8. 지원 과목 및 문제 유형

### 수능 과목 지원 범위 (MVP: 국어·수학·영어)

> **MVP 범위**: 국어(독서/문학), 수학(미적분/확통/기하), 영어(독해/듣기) 3과목의 과목별 전용 프롬프트 최적화 완료.  
> DB 스키마(`suneung_types.json`)에는 28개 수능 유형 전체가 정의되어 있으며, 미최적화 과목은 기본 프롬프트로 폴백 동작.  
> 사회탐구·과학탐구·제2외국어는 v1.1에서 과목별 프롬프트 추가 예정.

CLAIQ는 수능 전 영역을 커버합니다. 교강사가 수업을 등록할 때 해당 과목을 선택하면, 그 과목의 수능 출제 패턴에 맞는 프롬프트가 자동으로 적용되어 문제가 생성됩니다.

| 영역 | 포함 과목 |
|------|-----------|
| 국어 | 화법과 작문, 언어와 매체, 독서, 문학 |
| 수학 | 수학 I, 수학 II, 확률과 통계, 미적분, 기하 |
| 영어 | 영어 독해, 영어 듣기 |
| 한국사 | 한국사 |
| 사회탐구 | 생활과 윤리, 윤리와 사상, 한국지리, 세계지리, 동아시아사, 세계사, 경제, 정치와 법, 사회·문화 |
| 과학탐구 | 물리학 I/II, 화학 I/II, 생명과학 I/II, 지구과학 I/II |
| 직업탐구 | 농업 기초 기술, 공업 일반, 상업 경제, 수산·해운 산업 기초, 인간 발달 |
| 제2외국어/한문 | 독일어, 프랑스어, 스페인어, 중국어, 일본어, 러시아어, 아랍어, 베트남어, 한문 |

### 문제 유형

교강사는 문제 생성 시 아래 세 가지 유형 중 하나를 선택할 수 있습니다.

| 유형 | 설명 | 주요 적용 과목 |
|------|------|----------------|
| 객관식 (5지선다) | 수능 기본 형태. 5개 보기 중 1개 정답 | 국어, 영어, 사탐, 과탐, 한국사 등 |
| 단답형 | 수학 주관식 형태. 숫자 또는 짧은 서술 정답 | 수학 |
| 혼합 | 객관식 + 단답형 동시 출제 | 전 과목 (교강사 재량) |

## 9. 포인트 시스템

학습 동기 부여를 위한 포인트 시스템으로, 꾸준히 참여하면 실질적인 보상을 받을 수 있습니다. 한 달 성실하게 이용 시 약 5,000포인트(= 5,000원 상당)가 적립됩니다.

### 포인트 적립 기준

| 행동 | 포인트 |
|------|--------|
| 일일 출석 | 20p |
| 문제 5개 이상 풀기 (참여 보상) | 10p |
| 정답 1개당 (난이도 A) | 10p |
| 정답 1개당 (난이도 B) | 15p |
| 정답 1개당 (난이도 C) | 25p |
| 5문제 전부 정답 퍼펙트 보너스 | 50p |
| AI Q&A 이용 | 10p |
| 7일 연속 스트릭 보너스 | 100p |
| 주간 목표 달성 | 150p |
| 월간 개근 (30일) | 500p |
| 30일 스트릭 달성 | 700p |
| 뱃지 획득 | 50~200p |

- A등급 5문제 전부 정답: 참여(10) + 정답(50) + 퍼펙트(50) = 110p
- C등급 5문제 전부 정답: 참여(10) + 정답(125) + 퍼펙트(50) = 185p
- 5문제 전부 오답: 참여(10) = 10p (포기 방지)

> 월 기준 예상 적립량: 일일 출석(20p × 30일) + 문제 풀기(30p × 20일) + AI Q&A(10p × 20회) + 주간 목표 달성(150p × 4주) + 월간 개근(500p) ≈ **2,300 ~ 5,000p**

### 포인트 교환 기준

| 포인트 | 교환 혜택 |
|--------|-----------|
| 5,000p | 5,000원 기프티콘 (올리브영, 편의점, 배달앱 등) |
| 자유 설정 | 학원비 할인 쿠폰 (운영자가 금액·조건 직접 설정 가능) |

> 기프티콘 브랜드 목록 및 학원비 할인 쿠폰 조건은 운영자 대시보드에서 자유롭게 설정할 수 있습니다.

## 10. 기술 구조

### 전체 아키텍처

```
[수강생/교강사/운영자 브라우저]
         │
    React 19 (Vercel)
         │ REST API
    Node.js + Express (Render)
         │
    ┌────┴──────────────┐
    │                   │
PostgreSQL            OpenAI API
(pgvector 확장)       ├── GPT-4 (문제 생성 / 답변 / 로드맵 생성)
                      ├── Whisper (STT)
                      └── Embedding (벡터화)
```

### AI 파이프라인 상세

#### 문제 자동 생성 파이프라인

```
교강사 녹음 파일 업로드
        ↓
Whisper API → 강의 텍스트(Transcript)
        ↓
OpenAI Embedding → 벡터 변환 → pgvector 저장 (RAG 지식 베이스)
        ↓
GPT-4 프롬프트
  - 과목별 수능 출제 패턴 시스템 프롬프트 자동 적용
  - 문제 유형 선택: 객관식(5지선다) / 단답형 / 혼합
  - 난이도 A(기본) / B(표준) / C(심화) 각각 생성
        ↓
문제 후보 생성
        ↓
교강사 검증 화면 (승인 / 수정 / 반려) ← Human-in-the-Loop
        ↓
승인된 문제 DB 저장 → 수강생 출제
```

#### 수능 유형 매핑 파이프라인

```
강의 텍스트(Transcript) 입력
        ↓
GPT-4 분석
  - 수능 출제 기준 유형 DB와 대조 (과목별 세부 유형 목록 사전 정의)
  - 예: 국어 독서 → [사실적 이해 / 추론 / 빈칸 추론 / 적용·감상 / ...]
        ↓
해당 수업의 수능 유형 태그 자동 부착 (1개 이상 복수 매핑 가능)
        ↓
생성 문제에 수능 유형 메타데이터 포함 저장
        ↓
수강생 풀이 결과 → 수능 유형별 정답률 누적 집계
        ↓
취약 수능 유형 식별 → 로드맵·미니 모의고사 생성에 활용
```

#### 수능 D-day 역산 로드맵 생성 파이프라인

```
입력값 수집
  - 수능 D-day (고정 날짜 또는 사용자 설정)
  - 학생 수능 유형별 정답률 (누적 데이터)
  - 향후 수업 일정 (교강사 등록 일정)
        ↓
GPT-4 전략 분석
  - 취약 유형 우선순위 산출 (정답률 낮은 순 + 수능 출제 빈도 가중치)
  - 남은 일수 대비 커버 가능한 유형 수 계산
  - 수업 일정과 연계한 학습 순서 배치
        ↓
개인별 로드맵 생성
  - "수능까지 N일 / 이번 주 집중 유형: X / 학습 순서: [강의A → 문제B → 강의C]"
        ↓
매주 자동 재계산 (풀이 데이터 갱신 반영)
        ↓
수강생 화면에 로드맵 표시
```

#### 개인별 미니 모의고사 생성 파이프라인

```
학생 취약 수능 유형 목록 조회
        ↓
수능 출제 패턴 DB 참조
  - 유형별 문항 수 배분 (실제 수능 비율 기준)
  - 시간 제한·배점 설정
        ↓
GPT-4 문항 생성
  - 취약 유형 집중 편성 (약점 유형 비중 70% 이상)
  - 기출 유형 패턴에 맞는 지문·선지 구성
        ↓
미니 모의고사 패키지 완성 (문항 + 시간 + 배점)
        ↓
수강생 풀이 → 결과 수집
        ↓
수능 유형별 분석 리포트 자동 생성 및 제공
```

#### RAG 기반 Q&A 파이프라인

```
수강생 질문 입력
        ↓
OpenAI Embedding → 질문 벡터화
        ↓
pgvector 유사도 검색 (해당 교강사 강의 범위 필터)
        ↓
상위 K개 강의 청크 추출 (Retrieval)
        ↓
GPT-4 → 검색된 강의 내용 기반 답변 생성 (Augmented Generation)
        ↓
수강생에게 응답 반환
```

#### 약점 분석 및 추천 파이프라인

```
수강생 문제 풀이 데이터 누적
        ↓
수능 유형별 정답률 분석 → 취약 유형 식별
        ↓
취약 유형 키워드 → OpenAI Embedding → 벡터화
        ↓
pgvector 유사도 검색
        ↓
추천 강의 + 추천 문제 목록 반환
```

### 기술 스택 요약

| 영역 | 기술 |
|------|------|
| 프론트엔드 | React 19, Vite 7, Vercel |
| 백엔드 | Node.js, Express, Render |

## 11. 기대 효과

### 교강사 업무 효율화

- 문제 제작 시간 **80% 단축**: 수업당 평균 3시간 → 15분 (생성 + 검토 포함)
- 개별 질문 응대 부담 **60% 감소**: AI 1차 응대가 반복 질문의 대부분을 처리
- 강의 콘텐츠 재활용률 향상: 녹음 한 번으로 문제·RAG 지식 베이스·정리자료 동시 생성

### 수강생 학습 성과

- 취약점 조기 발견: 단원 종료 전 수능 유형별 약점 파악 → 평균 오답률 감소 기대
- 자기주도 학습 강화: 24시간 AI Q&A + D-day 로드맵 + 맞춤 추천으로 자습 시간 효율 증가
- 학습 지속성 향상: 스트릭·뱃지·포인트 시스템으로 중도 이탈률 감소 기대
- 실전 감각 향상: 개인 맞춤 미니 모의고사로 수능 형식에 익숙해지는 반복 훈련

### 운영자·학원 경영 개선

- **이탈 예측 사전 개입**: 퇴원 통보 전 위험 신호 감지 → 이탈률 20~30% 감소 목표
- 데이터 기반 강사 피드백: 강의별 이해도 통계로 커리큘럼 개선 사이클 단축
- 학부모 신뢰도 제고: 정기 성취 리포트 발송으로 재등록률 향상 및 구전 마케팅 효과

### 플랫폼 확장성

- 현재: 수능 준비 중소 학원 특화 (수능 전 8개 영역 지원)
- 단기 확장: 검정고시 과정 및 내신 대비 기능 추가
- 장기 확장: 대형 학원 체인, 온라인 강의 플랫폼과의 연동 가능

포인트·뱃지·스트릭 시스템은 단순 부가 기능이 아닌, 수능 학원 수강생의 특성을 반영한 행동설계 원칙에 기반합니다.

### 7종 뱃지 설계 의도

| 뱃지 | 발동 조건 | 설계 근거 |
|------|----------|----------|
| QUIZ_FIRST | 첫 문제 풀기 | 온보딩 완료율 향상 — 첫 행동 완료 시 즉각 보상으로 초기 이탈 방지 |
| ATTENDANCE_STREAK (7일) | 7일 연속 접속 | 수능 학원 주간 수업 주기(월~금 + 주말 자습)와 일치하는 단위로 설계 |
| ATTENDANCE_STREAK (30일) | 30일 연속 | 수능까지 남은 월 단위 목표와 연동 — 장기 학습 지속 동기 |
| WEAK_MASTERY | 취약 유형 정답률 향상 | 학원 이탈 주원인인 "성적 미향상 좌절감" 직접 해소. 취약점 극복 경험이 자기효능감 형성 |
| MINI_EXAM_COMPLETE | 첫 모의고사 완료 | 20분 집중 경험 — 실제 시험 환경 적응력 형성 |
| QA_ACTIVE | AI Q&A 5회 이상 | 질문 심리적 장벽 해소 후 능동 학습자 전환 유도 |
| REPORT_DELIVERED | 성취 리포트 발송 | 학부모 공유 → 외부 인정 욕구 충족 → 지속 동기 |
