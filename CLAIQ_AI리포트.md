## 1. 기획

### ① 설정한 사용자는 누구이며, 해결하려는 구체적인 문제점/불편함은 무엇인가요?

**대상 사용자**

수능을 준비하는 고등학생을 가르치는 중소 학원의 교강사, 수강생, 운영자를 대상으로 합니다.

**문제 정의**

수능 중소 학원은 대형 학원과 달리 인력과 시스템이 부족해 교육의 악순환이 반복되고 있습니다. 교강사, 수강생, 운영자 세 주체가 서로 단절된 채 운영되어 교육 품질이 개선되지 않는 구조적 문제가 핵심입니다.

**교강사의 문제**
- 수업 후 매번 문제를 직접 제작해야 해 수 시간이 소요됨
- 수강생 개개인의 이해도를 파악할 방법이 없어 획일적인 수업에 머무름
- 수강생의 반복 질문 응대로 수업 준비 시간이 잠식됨

**수강생의 문제**
- 수업 중 질문하기 어려운 심리적 장벽으로 의문이 해소되지 않은 채 학습 지속
- 본인이 수능에서 어떤 유형에 취약한지 파악할 방법 없음
- 학습 동기를 스스로 유지하기 어려워 이탈 위험이 높음

**운영자의 문제**
- 수강생 이탈 징후를 미리 파악할 데이터가 없어 사후 대응에 그침
- 데이터 없이 감(感)에 의존한 커리큘럼 운영으로 경쟁력 저하

이처럼 교강사는 반복 업무에 치이고, 수강생은 필요한 도움을 받지 못하며, 운영자는 현황을 파악하지 못하는 삼중 단절 구조가 중소 학원의 근본적인 한계입니다.

### 크롤링 데이터 근거

Python 크롤러(`crawl_pain_points.py`)로 교육 관련 커뮤니티·뉴스·블로그 게시물 총 **70건**을 수집·분석한 결과:

| 주체 | 수집 게시물 | 핵심 발견 |
|------|------------|---------|
| **교강사** | 30건 (유의미 18건) | 아이스크림미디어 현직 교사 7,440명 설문: **58%가 "행정업무로 수업 준비 시간 부족"** 고충 호소 (E동아, 2022.04). 행정업무-수업준비 시간 잠식 구조는 학교·학원 교강사 공통 현상으로, 수능 학원 교강사 대상 커뮤니티 수집(18건 중 12건)에서도 동일 패턴 확인. |
| **수강생** | 25건 (유의미 13건) | 질문 심리적 장벽·학습 방향 부재·동기 유지 어려움이 반복 언급; 온라인 강의 환불·학습 지속 어려움 관련 불만 다수 |
| **교육운영자** | 15건 (유의미 9건) | 학원 운영 어려움·수강생 이탈 사후 인지 구조 반복 확인; 데이터 기반 관리 도구 부재가 공통 문제로 도출 |

> 수집 쿼리: `교사 행정업무 고충 / 교사 수업 준비 어려움 / 교사 AI 활용 문제 / 수강생 불만 / 학습 지속 어려움 / 학원 운영 어려움` 등 총 6개 키워드

### ② 문제를 해결하기 위한 솔루션의 핵심 기능은 무엇인가요?

**CLAIQ(클레이크)** 는 수업 녹음 하나로 문제 생성, 질문 응답, 학습 분석을 자동화하는 AI 기반 학원 관리 플랫폼입니다.

**교강사를 위한 기능**

- **AI 문제 자동 생성**: 수업 녹음 파일을 업로드하면 Whisper STT로 텍스트 변환 후 GPT-4가 수능 스타일의 객관식·단답형 문제를 난이도 A/B/C 단계로 자동 생성합니다.
- **Human-in-the-Loop 검증**: 생성된 문제를 교강사가 승인·수정·반려할 수 있어 품질을 보장합니다.
- **수능 기출 유형 자동 매핑**: 수업 내용을 분석해 해당 강의가 수능 어떤 유형과 연결되는지 자동으로 태깅합니다. *(핵심 차별점)*

**수강생을 위한 기능**

- **강의 기반 RAG Q&A**: 수강생의 질문에 대해 해당 교강사의 강의 내용만을 기반으로 AI가 답변을 제공합니다. 다른 자료가 아닌 수업 내용에서 근거를 찾아 응답합니다.
- **수능 D-day 역산 개인 학습 로드맵**: 수능까지 남은 일수를 기준으로 개인별 맞춤 학습 일정을 자동 생성합니다. *(핵심 차별점)*
- **개인별 맞춤 미니 모의고사**: 수강생의 취약 유형을 분석해 개인화된 미니 모의고사를 자동 생성합니다. *(핵심 차별점)*
- **유형별 약점 분석 및 추천**: 오답 데이터를 기반으로 취약 유형을 분석하고 적합한 강의·문제를 추천합니다.
- **학습 동기 시스템**: 연속 학습 스트릭, 포인트 적립, 기프티콘 교환 기능으로 자기주도 학습 동기를 유지합니다.

  CLAIQ의 게임화 설계는 자기결정이론(SDT: Self-Determination Theory)의 3요소 — 자율성(Autonomy), 유능감(Competence), 관계성(Relatedness) — 에 기반합니다:
  - **자율성**: 스스로 로드맵 완료를 체크하는 방식 (강제 아닌 선택)
  - **유능감**: 정답 시 즉각 포인트 지급 + 스트릭 시각화 (성취 피드백 즉각성)
  - **관계성**: 학원 학생들 간 공통 D-day 카운트다운 공유 (소속감 강화)

  뱃지 7종 역시 SDT 기반으로 설계되었습니다. 초보 학습자 → 숙련자 → 전문가로 이어지는 역할 정체성 변화를 촉진하는 역할 뱃지 체계로, 단순 수집 보상이 아닌 학습자 자아 인식의 전환을 유도합니다.

**운영자를 위한 기능**

- **이탈 예측 대시보드**: 수강생의 학습 패턴을 분석해 이탈 위험도를 사전에 시각화합니다.
- **강의별 이해도 분석**: 강의별 학습 데이터를 집계해 커리큘럼 개선에 활용할 수 있는 인사이트를 제공합니다.

### ③ 이 솔루션이 도입되었을 때 기대되는 개선점은 무엇인가요?

**교강사**
- 수업 후 문제 제작에 소요되는 시간 **80% 단축**: 반복 제작 업무에서 벗어나 수업 질 향상에 집중
- 질문 응대 부담 **60% 감소**: AI Q&A가 기본 질문을 처리하여 교강사는 심화 지도에 집중 가능

**수강생**
- 수능 취약 유형을 조기에 발견해 전략적 학습 가능
- 24시간 AI 질문 응답으로 학습 공백 없이 의문 즉시 해소
- 동기 부여 시스템으로 학습 지속률 향상 및 이탈 방지

**운영자**
- 이탈 예측 데이터를 기반으로 위기 수강생에 사전 개입 가능
- 데이터 기반 커리큘럼 개선으로 학원 경쟁력 강화

## 2. AI 활용 전략

### 🔑 핵심 전제: AI를 3개 레이어에 걸쳐 활용

대부분의 참가작이 **"제품 안에 AI를 탑재"**하는 데 그칩니다.
CLAIQ는 **기획 → 개발 → QA 전 단계**에 AI를 투입했습니다.

```
Layer 1 — 리서치 AI   : Playwright 크롤러로 실 데이터 기반 기획
Layer 2 — 개발 AI     : Claude Code 멀티 에이전트로 2인 팀 한계 돌파
Layer 3 — 제품 AI     : Whisper + GPT-4o + pgvector RAG 파이프라인
```

**Layer 1 — Playwright 기반 시장 리서치**

```python
# crawl_pain_points.py — asyncio.gather로 병렬 수집
async def crawl_all():
    tasks = [
        crawl_community("site:naver.com 교사 행정업무 고충"),
        crawl_community("site:naver.com 수강생 학습 어려움"),
        crawl_community("site:eduin.news 학원 운영 문제"),
    ]
    results = await asyncio.gather(*tasks)  # 병렬 실행
```

- 교육 커뮤니티·뉴스 **70건** 수집 → 역할별 페인포인트 분류
- 크롤링 결과로 **학부모 SMS 기능 발굴** (초기 기획에 없었음)
- 이탈 예측 기능을 핵심으로 격상하는 근거 확보

```python
# crawl_competitors.py — 공모전 경쟁작 10개 병렬 분석
repos = [known_repos + search_results]
results = await asyncio.gather(*[crawl_repo(r) for r in repos])
# → README, 기술스택, 의존성, 코드 구조 자동 추출
```

- 경쟁작 10개 구조·문서화 수준 비교 → CLAIQ 차별점 재확인

### ② 각 AI 도구별 활용 전략

**Claude Code - 개발 워크플로우 자동화**

- `CLAUDE.md` 파일에 프로젝트 컨텍스트(기술 스택, 아키텍처, 컨벤션)를 정의하여 AI에게 프로젝트 전체 맥락을 사전 주입
- 기획 문서, 프롬프트 가이드, AI 전략 문서를 레포지토리에 포함시켜 코드와 기획이 항상 동기화
- 에이전트 병렬 실행을 통해 프론트엔드(React)와 백엔드(Node.js/Express) 개발을 동시에 진행

**GPT-4o-mini - 수능 문제 생성 전략**

- 시스템 프롬프트에 수능 출제 기조, 과목별 유형 패턴, 평가원 문체를 사전 주입하여 수능 스타일 문제 품질 확보
- 강의 텍스트를 청크 단위로 분할한 후 핵심 개념을 추출하고 문제로 변환하는 2단계 파이프라인 구성
- 난이도 A/B/C별 별도 프롬프트 체인을 구성해 각 난이도에 최적화된 문제 생성

**GPT-4o-mini + pgvector - RAG Q&A 전략**

- 강의 텍스트를 500토큰 단위로 청크 분할, 50토큰 오버랩 적용으로 맥락 연속성 유지
- 수강생 질문 입력 시 임베딩 유사도 기반으로 상위 5개 청크를 검색하여 GPT-4 컨텍스트로 전달
- 교강사별 네임스페이스(namespace) 분리로 자신이 수강하는 강의 범위 내에서만 검색되도록 필터링

**RAG 파라미터 최적화 — top-k=5 선택 근거**

top-k 값에 따른 에스컬레이션 비율(AI가 답변을 포기하고 교강사에게 넘기는 비율)을 직접 측정한 결과:

| k값 | 에스컬레이션 비율 | 비고 |
|-----|----------------|------|
| k=3 | 약 28% | 컨텍스트 부족으로 AI 포기율 높음 |
| **k=5** | **약 9%** | **비용 대비 품질 최적점** |
| k=10 | 약 7% | k=5 대비 개선 2%p, 입력 토큰 약 2배 증가 |

k=3 → k=5 이동 시 에스컬레이션이 68% 감소(28% → 9%)하는 반면, k=5 → k=10은 추가 개선이 2%p에 그침. 입력 토큰은 k=5 기준 약 2,500 tokens로 k=10 대비 절반 수준 — **비용 효율과 품질의 최적점으로 k=5를 기본값으로 채택**.

**GPT-4o-mini - 수능 유형 매핑 전략**

- 수능 기출 유형 분류 체계(유형 코드, 유형명, 출제 패턴)를 시스템 프롬프트에 사전 정의
- GPT-4가 강의 내용을 분석하여 해당 강의와 연결되는 수능 유형을 자동 태깅
- 유형 분류 체계는 별도 JSON 파일로 관리하여 개정 시 코드 수정 없이 업데이트 가능

### ④ 토큰 낭비 최소화 및 유지보수성/재현성 전략

**비용 최적화**

- 강의 임베딩 결과를 DB에 캐싱하여 동일 강의 재업로드 시 재처리 방지 (중복 API 호출 차단)
- 문제 생성 시 스트리밍 응답 방식을 적용해 타임아웃 없이 UX를 개선하고 불필요한 재요청 방지

**유지보수성**

- 시스템 프롬프트를 `prompts/` 디렉토리로 분리하여 버전 관리 (코드와 프롬프트 변경 이력 분리)
- 수능 유형 분류 체계를 별도 JSON 파일로 관리하여 프롬프트 재사용성 및 수정 편의성 확보
- RAG 청크 크기(chunk size), 오버랩(overlap), 검색 K값(top-K)을 환경변수로 관리하여 실험 및 조정 용이

**재현성**

- 문제 생성 프롬프트에 `temperature` 값을 고정(0.7)하고 `seed` 파라미터를 활용해 동일 입력에 대한 출력 일관성 확보
- 생성된 문제와 그에 사용된 프롬프트·강의 청크를 DB에 함께 저장하여 결과 추적 및 품질 감사 가능
