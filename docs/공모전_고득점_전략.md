## 심사 기준별 고득점 전략

### 1. 기술적 완성도

**수상작 공통점:**
- 핵심 기능이 실제로 동작하는 배포 URL 제공 (프로토타입 X)
- 에러 처리, 인증, 보안까지 마무리된 상태
- README에서 바로 설치 → 실행 가능한 수준의 문서

**CLAIQ 현황:**
- ✅ 실제 배포 URL (claiq.vercel.app)
- ✅ JWT 인증, Rate Limiting, Helmet 보안
- ✅ 5개 AI 모듈 전체 구현 (STT, 문제생성, RAG, 로드맵, 모의고사)
- ⚠️ 기능 테스트 완료 필요

**추가로 하면 좋은 것:**
- `GET /api/health` 엔드포인트 → UptimeRobot 모니터링 연결 (이미 완료)
- 핵심 기능 동작 스크린샷을 README에 추가

### 2. AI활용 능력 및 효율성

**수상작 공통점:**
- AI를 단순 챗봇이 아닌 **파이프라인**으로 활용
- AI 협업 과정(프롬프트, 에이전트, 결정 기록)을 문서로 남김
- CLAUDE.md + AGENTS.md + ADR이 있는 레포가 없는 레포보다 문서화 수준 높음

**배곡(baegok)이 잘 한 것:**
```
.claude/commands/     # 슬래시 커맨드 정의
.agents/skills/       # 역할별 에이전트 스킬
docs/decisions/ADR-*  # 9개 기술 결정 기록
```

**CLAIQ 현황:**
- ✅ CLAUDE.md (AI 협업 지침)
- ✅ AGENTS.md (10개 전문 에이전트 활용 방식)
- ✅ AI_협업_기록.md (작업 분기별 기록)
- ✅ ADR 3개 (모델 선택, RAG 아키텍처, HITL)
- ✅ docs/ai_pipeline_architecture.md

**추가로 하면 좋은 것:**
- AI와 주고받은 프롬프트 전략 간략히 문서화
- 기능 테스트 후 AI_협업_기록.md에 "테스트 및 버그 수정" 분기 추가

### 3. 기획력 및 실무 접합성

**수상작 공통점:**
- 추상적 기능 X → **실제 사용자 고통**에서 출발
- 타깃이 명확 (외국인 근로자 / 환아 / 수능 수강생)
- 상업적 실현 가능성 언급 (수익 모델, 확장 계획)

**LinkON이 잘 한 것:**
```markdown
| 문제점 | 현장 상황 |
| 현장 교육 체계 부재 | 전담 인력 부족으로 구전 교육에 의존 → 불량 발생 |
| 휴먼 에러의 악순환 | 긴급 수칙이 실시간 전달 안 됨 |
| 소통 단절과 이탈 | 언어 장벽으로 고충 보고 누락 → 숙련 인력 이탈 |
```
→ 문제를 표로 구체화하고 해결책을 명확히 대응

**CLAIQ 현황:**
- ✅ README에 4개 주체(교강사/수강생/운영자/학부모) 고통 → 해결책 매핑
- ✅ 실제 수능 기출 유형 기반 문제 생성
- ⚠️ 수익 모델(SaaS 구독, 포인트 쿠폰 연동) 기획안에만 있고 README에 미반영

### 4. 창의성

**수상작 공통점:**
- 기존에 없던 조합 또는 틈새 시장 공략
- AutoRAG: "AutoML 개념을 RAG에 적용" → 새로운 카테고리 창출
- SPOT!: "K-드라마 촬영지 관광" → 트렌드(셋-제팅) 공략

**CLAIQ 차별점으로 어필할 것:**
- **수능 D-day 역산 로드맵**: 시험일 기준 역산해 우선순위 자동 결정 → 타 학습 플랫폼에 없음
- **Human-in-the-Loop 문제 검수**: AI 생성 → 교강사 검수 → 학생 출제 3단계 품질 보장
- **이탈 예측 대시보드**: 학원 운영자 관점의 AI 활용 (학생 대상 AI와 차별화)
- **3개 역할 통합**: 운영자/교강사/수강생이 하나의 플랫폼에서 연결
