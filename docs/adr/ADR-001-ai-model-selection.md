# ADR-001: AI 모델 선택

## 상태

**승인됨** - 2026-04-07

---

## 맥락

CLAIQ의 핵심 기능인 수능 문제 자동 생성에 사용할 AI 모델을 결정해야 했습니다.

개발·테스트 단계에서는 반복적인 API 호출이 많고 비용 소모가 크기 때문에,
**기본값을 저렴한 모델로 설정하되 프로덕션 전환 시 업그레이드 가능한 구조**가 필요했습니다.

### 검토된 선택지

| 모델 | 입력 비용 (1M 토큰) | 출력 비용 (1M 토큰) | 비고 |
|------|---------------------|---------------------|------|
| gpt-4o | $2.50 | $10.00 | 최고 품질 |
| **gpt-4o-mini** | **$0.15** | **$0.60** | **97% 저렴, 선택** |
| gpt-3.5-turbo | $0.50 | $1.50 | JSON 구조화 불안정 |

---

## 결정

**기본값: `gpt-4o-mini`**
**고품질 전환: `gpt-4o`** (`.env`에서 주석 해제로 즉시 전환 가능)

```env
OPENAI_MODEL_CHAT=gpt-4o-mini
# OPENAI_MODEL_CHAT=gpt-4o  # 프로덕션 고품질 옵션
```

---

## 근거

### 1. 개발·테스트 단계 비용 최소화

강의 업로드 → 문제 생성 파이프라인을 반복 테스트할 때 비용이 누적됩니다.
gpt-4o-mini 기본값으로 테스트 비용을 97% 절감하면서 기능 검증을 진행했습니다.

**강의 1개 문제 생성 기준 추정:**
- gpt-4o: 약 37원
- gpt-4o-mini: 약 2원

### 2. Human-in-the-Loop가 품질 보완

gpt-4o-mini가 생성한 문제는 교강사 검수(승인/수정/반려)를 통과해야 학생에게 출제됩니다.
모델 품질이 다소 낮더라도 사람의 검수가 최종 품질을 보장하므로 기본값으로 충분합니다.

### 3. 전환 비용 0

모델명이 환경변수 한 줄로 관리되므로, 프로덕션 배포 시 gpt-4o로 전환해도
코드 변경 없이 즉시 적용됩니다.

---

## 결과

- 개발 전체 과정에서 AI API 비용 최소화
- 기능 검증과 품질 보장을 분리하는 구조 확립 (모델 품질 + 교강사 검수)
- 프로덕션 전환 시 환경변수 1줄 변경으로 업그레이드 가능

---

## 트레이드오프

- gpt-4o-mini 생성 문제는 교강사 수정이 필요한 경우가 gpt-4o 대비 많을 수 있음
- 이는 Human-in-the-Loop 구조로 수용 가능한 수준으로 판단
