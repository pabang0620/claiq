# ADR-003: Human-in-the-Loop 설계

## 상태

**승인됨** - 2026-04-07

---

## 맥락

AI가 생성한 수능 문제를 학생에게 제공하는 방식을 결정해야 했습니다. AI가 생성한 콘텐츠를 즉시 학생에게 노출할지, 아니면 교강사의 검수를 거치도록 할지에 대한 선택이었습니다.

### 핵심 질문

> "AI가 문제를 잘 만들면 굳이 교강사 검수가 필요한가?"

이 질문은 기술적 효율성의 문제가 아니라 **교육 플랫폼으로서의 신뢰성과 책임** 문제입니다.

### 검토된 선택지

#### 선택지 1: 완전 자동화 (검수 없음)

AI가 생성한 문제를 즉시 학생에게 제공.

**장점:**
- 업로드 즉시 문제 제공 (교강사 대기 불필요)
- 운영 효율 극대화

**단점:**
- AI 오류 문제가 학생에게 그대로 노출
- 잘못된 정답, 오해를 유발하는 선지 등 교육적 피해 가능
- 플랫폼 신뢰도 하락 시 회복 어려움
- 교강사가 플랫폼에서 역할을 잃어 참여 동기 감소

#### 선택지 2: 교강사 검수 필수 (승인/수정/반려)

AI가 생성한 문제 초안을 교강사가 검토 후 승인/수정/반려.

**장점:**
- 교육 콘텐츠 품질 보장
- 교강사의 전문성이 최종 관문으로 작동
- 플랫폼 신뢰도 유지
- 교강사가 플랫폼 핵심 역할 유지 → 참여 동기 보장

**단점:**
- 강의 업로드 후 문제 제공까지 교강사 검수 시간 소요
- 교강사 부담 증가 가능성

#### 선택지 3: 신뢰도 기반 자동 공개

AI 생성 문제의 신뢰도 점수가 임계값을 넘으면 자동 공개, 낮으면 검수 요청.

**장점:**
- 고품질 문제는 즉시 제공
- 검수 부담 분산

**단점:**
- 신뢰도 점수의 신뢰성 자체를 검증하기 어려움
- "AI가 판단한 고품질"이 실제 수능 유형에 맞는지 보장 불가
- 자동 공개된 오류 문제 발생 시 책임 소재 불명확

---

## 결정

**교강사 검수 필수 (승인/수정/반려) 채택**

구체적인 검수 워크플로우:

```
AI 문제 생성 완료
       │
       ▼
교강사 검수 대기 상태 (pending)
       │
       ▼
교강사 검수 인터페이스에서 문제 확인
       │
  ┌────┴────────────────┐
  ▼                     ▼
승인 (approved)       반려 (rejected)
내용 직접 수정 후
approved 처리 가능
  │
  ▼
학생에게 노출
```

검수 인터페이스 제공 기능:
- 문제 전체 보기 (문항, 선지, 정답, 해설)
- 인라인 수정 UI: v1.1 로드맵 (MVP 범위 외)
- 정답 변경
- 난이도/유형 재분류
- 반려 시 사유 입력 (AI 재생성 피드백으로 활용)

## 근거

### 1. 교육 신뢰도는 한번 손상되면 회복이 어렵다

교육 플랫폼에서 잘못된 문제가 학생에게 제공되었을 때의 피해는 단순한 기술적 오류와 다릅니다.

- 학생이 틀린 정보를 학습하면 수능에 직접 영향
- "CLAIQ 문제는 오류가 있다"는 인식이 퍼지면 플랫폼 전체 신뢰 붕괴
- 교육 콘텐츠의 오류는 다른 분야보다 법적/윤리적 책임이 무거움

gpt-4o-mini가 생성한 문제는 형식적으로는 그럴듯하지만, 교육 관점에서 검수가 필요한 오류가 포함될 수 있습니다. 예를 들어:
- 정답이 2개인 문제
- 실제 수능 유형과 맞지 않는 문제 형식
- 오개념을 심어주는 선지 표현

이러한 문제들은 교강사만이 식별할 수 있습니다.

### 2. AI 오류 방지 - 발생 가능한 오류 유형

gpt-4o-mini가 생성한 문제에서 교강사 검수를 통해 수정이 필요할 수 있는 유형:

| 오류 유형 | 심각도 |
|-----------|--------|
| 선지 표현 모호 (정답이 2개로 해석 가능) | 높음 |
| 수능 유형 형식 불일치 | 중간 |
| 해설 부정확 (정답은 맞으나 설명이 틀림) | 중간 |
| 근거 본문과 정답 불일치 | 높음 |
| 난이도 분류 부적절 | 낮음 |

이러한 오류들은 교강사 검수 없이 자동 공개 방식을 채택할 수 없는 근거입니다. 실제 발생 빈도는 운영 후 측정할 예정입니다.

### 3. 교강사 권한 유지 - 플랫폼 핵심 가치

CLAIQ의 포지셔닝은 "AI가 교강사를 대체하는 플랫폼"이 아니라 **"AI가 교강사의 생산성을 높이는 플랫폼"**입니다.

교강사가 문제 검수 역할을 유지함으로써:
- 자신의 강의 콘텐츠에 대한 통제권 보유
- AI 생성 문제 품질에 대한 전문적 판단
- 학생과의 관계에서 "내 문제"라는 ownership 유지
- 반려 피드백이 AI 개선 데이터로 축적

교강사를 단순 콘텐츠 제공자가 아닌 **AI-assisted creator**로 포지셔닝하는 것이 장기적으로 플랫폼 생태계를 건강하게 유지합니다.

### 4. 검수 부담 최소화 설계

검수 필수를 결정하되, 교강사의 부담을 최소화하는 UX를 설계했습니다.

**효율적 검수를 위한 설계:**
- 문제 목록 한눈에 보기 (리스트 뷰)
- 빠른 승인을 위한 키보드 단축키 (승인: Enter, 반려: Delete)
- 수정이 필요한 문제만 필터링
- AI가 신뢰도 낮다고 표시한 문제를 상단에 배치
- 일괄 승인 기능 (전체 문제 검토 후 한번에 승인)

UX 설계 목표: 일괄 승인 및 키보드 단축키를 활용해 검수 부담을 최소화 (실제 소요 시간은 운영 후 측정 예정)

## 결과

### 품질 보장 체계 확립

Human-in-the-Loop 설계로 다음이 보장됩니다:

1. **오류 문제 차단**: 교강사 검수를 통과하지 않은 문제는 학생에게 절대 노출되지 않음
2. **유형 적합성 보장**: 실제 수능 유형에 맞는지 전문가 확인
3. **교강사 책임 명확화**: 공개된 문제의 최종 책임은 검수한 교강사에게

### 교강사 참여도 유지

검수 역할을 통해 교강사가 플랫폼에서 지속적으로 활동할 이유가 생깁니다.

- 강의 업로드 → AI 문제 생성 → 검수 → 학생 제공의 루프
- 반려 피드백 축적 → AI 품질 지속 향상
- 교강사가 수정한 문제 패턴이 향후 파인튜닝 데이터로 활용 가능

### 운영 지표 목표 (운영 후 측정 예정)
- 문제 검수 완료율: 업로드 후 24시간 내 목표 설정 예정
- 평균 검수 시간: 운영 후 측정
- 승인률: AI 품질 개선 지표로 운영 후 추적

### 향후 고도화 방향
- 교강사 피드백 패턴을 분석하여 AI 프롬프트 자동 개선
- 특정 교강사가 자주 수정하는 패턴을 해당 강의에 특화된 프롬프트로 반영
- 신뢰도 높은 문제 유형(예: TYPE_A 사실 확인)에 한해 자동 공개 확대 검토 (6개월 후)

## MVP 범위 결정 (트레이드오프)

인라인 수정 기능을 MVP에서 제외한 이유:
- 구현 예상 소요: 추가 2일
- 대신 배치: Playwright 자동화 QA, 배포 안정화, 시드 데이터 완성
- 대안: 교강사가 반려 후 재생성 요청 또는 사본 복사 후 외부 편집 → 재업로드
- v1.1 우선순위: 인라인 수정 → 마크다운 지원 → 이미지 삽입 순으로 단계적 구현 예정
