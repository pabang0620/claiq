## A. 인증 (Auth)

### A01 — 이메일 형식 유효성
- **조작**: 로그인 폼에 `jg@test.t` 입력 후 로그인 클릭
- **기대**: 폼 제출 안 됨, `'올바른 이메일 형식으로 입력해 주세요'` 인라인 에러
- **검증**: `page.locator('text=올바른 이메일 형식').isVisible()`

### A02 — 비밀번호 오류 응답
- **조작**: `admin@claiq.kr` + 틀린 비밀번호로 로그인
- **기대**: Alert/Toast에 `'이메일 또는 비밀번호가 올바르지 않습니다'`
- **검증**: 알림 텍스트 확인, URL 변화 없음

### A03 — 운영자 데모 로그인 전체 플로우
- **조작**: 데모 버튼 `'정민석'` 클릭
- **기대**: `/operator` 리다이렉트, 사이드바에 운영자 메뉴 표시, 헤더에 `'정민석'` 표시
- **검증**: URL, 이름 텍스트

### A04 — 로그아웃 confirm 동작
- **조작**: 로그아웃 버튼 클릭 → confirm 다이얼로그 `'취소'` 클릭
- **기대**: 로그아웃 안 됨, 그대로 대시보드
- **검증**: URL 유지

### A05 — 로그아웃 후 인증 초기화
- **조작**: 로그인 → 로그아웃 confirm → `/operator` 접근 시도
- **기대**: `/login`으로 리다이렉트
- **검증**: URL

### A06 — 회원가입 폼 유효성 (운영자)
- **조작**: 운영자 역할 선택 → 이름/학원명 비워두고 제출
- **기대**: 각 필드에 인라인 에러 메시지
- **검증**: `'이름을 입력하세요'`, `'학원 이름을 입력하세요'`

### A07 — 회원가입 비밀번호 확인 불일치
- **조작**: 비밀번호 `claiq1234`, 확인 `claiq5678`
- **기대**: `'비밀번호가 일치하지 않습니다'`
- **검증**: 인라인 에러

### A08 — 운영자 회원가입 → 학원코드 모달
- **조작**: 신규 운영자 계정 가입 (고유 이메일)
- **기대**: 가입 후 대시보드 이동, 학원코드 모달 표시
- **검증**: Dialog 존재, 코드 텍스트 non-empty

### A09 — 중복 이메일 가입 차단
- **조작**: `admin@claiq.kr`로 가입 시도
- **기대**: `'이미 사용 중인 이메일입니다'` Alert
- **검증**: 알림 텍스트

### A10 — 역할별 접근 제어
- **조작**: 수강생 로그인 → `/operator` 직접 접근 시도
- **기대**: `/unauthorized` 또는 `/login` 리다이렉트
- **검증**: URL

## O. 운영자 (Operator)

### O01 — 학원명 수정 및 저장
- **조작**: 학원 설정 → `#academy-name` 에 새 이름 입력 → 저장 버튼
- **기대**: Alert/Toast `'학원 정보가 저장됐습니다.'`
- **검증**: 토스트 텍스트, 페이지 리로드 후 변경 이름 유지

### O02 — 학원 코드 읽기 전용
- **조작**: 학원 설정 페이지 확인
- **기대**: 학원 코드가 `<span>` 또는 읽기 전용으로만 표시, 수정 불가
- **검증**: `input[name="code"]` 없음, font-mono span 존재

### O03 — 학원 소개 수정 저장
- **조작**: `#academy-desc` 에 텍스트 입력 → 저장
- **기대**: Toast 성공 메시지
- **검증**: 토스트 텍스트

### O04 — 쿠폰 생성 (퍼센트 할인)
- **조작**: 쿠폰명 `'테스트 10% 쿠폰'` + 퍼센트 + 할인값 `10` + 유효기간 `30` → 쿠폰 생성 버튼
- **기대**: Toast `'쿠폰이 생성됐습니다.'`, 목록에 새 쿠폰 추가
- **검증**: 토스트 텍스트, 쿠폰명 텍스트 목록에 존재

### O05 — 쿠폰 생성 (고정 금액)
- **조작**: 쿠폰명 `'5000원 할인'` + 고정 금액 + 할인값 `5000` → 생성
- **기대**: 목록에 `'5,000원 할인'` 텍스트 표시
- **검증**: 쿠폰 목록 텍스트

### O06 — 쿠폰 삭제 취소
- **조작**: 쿠폰 삭제 버튼 → confirm `'취소'`
- **기대**: 쿠폰 삭제 안 됨, 목록 유지
- **검증**: 쿠폰 개수 변화 없음

### O07 — 쿠폰 삭제 확인
- **조작**: 쿠폰 삭제 버튼 → confirm `'삭제'`
- **기대**: Toast `'쿠폰이 삭제됐습니다.'`, 목록에서 제거
- **검증**: 토스트 텍스트, 해당 쿠폰명 없음

### O08 — 학원 설정 저장 후 데이터 유지
- **조작**: 학원명 수정 → 저장 → 페이지 새로고침 (`/login` 후 재로그인)
- **기대**: 수정된 학원명 그대로 표시
- **검증**: `#academy-name` 값

### O09 — 회원 목록 전체 조회
- **조작**: 회원 관리 페이지 접근
- **기대**: 교강사 + 수강생 전체 목록 표시
- **검증**: 이름 텍스트 (이준혁, 김민준 등) 존재

### O10 — 회원 탭 필터 (교강사)
- **조작**: `'교강사'` 탭 클릭
- **기대**: 교강사만 표시, 수강생 없음
- **검증**: `'이준혁'` 존재, `'김민준'` 없음

### O11 — 회원 탭 필터 (수강생)
- **조작**: `'수강생'` 탭 클릭
- **기대**: 수강생만 표시, 교강사 없음
- **검증**: `'김민준'` 존재, `'이준혁'` 없음

### O12 — 회원 초대 모달 열기
- **조작**: `'멤버 초대'` 버튼 클릭
- **기대**: 모달 열림, 이메일 입력 필드 + 역할 선택 표시
- **검증**: `#invite-email` visible

### O13 — 회원 초대 — 존재하지 않는 이메일
- **조작**: 초대 모달 → 없는 이메일 입력 → 초대
- **기대**: `'해당 이메일의 사용자를 찾을 수 없습니다'` Alert
- **검증**: 알림 텍스트

### O14 — 회원 제거 confirm 취소
- **조작**: 멤버 제거 버튼 → confirm `'취소'`
- **기대**: 제거 안 됨, 멤버 목록 유지
- **검증**: 멤버 여전히 존재

### O15 — 리포트 목록 조회
- **조작**: 리포트 페이지 접근
- **기대**: 수강생 카드 목록 표시, 기존 리포트가 있으면 리포트 카드 표시
- **검증**: 수강생 이름 텍스트

### O16 — 리포트 생성
- **조작**: 수강생 카드에서 `'리포트 생성'` 버튼 클릭
- **기대**: 버튼 로딩 → 완료 → Toast `'리포트가 생성됐습니다.'` → 리포트 카드 추가
- **검증**: 토스트, 리포트 카드 존재

### O17 — SMS 발송 (시뮬레이션)
- **조작**: 리포트 카드 SMS 발송 버튼 클릭
- **기대**: Toast `'SMS가 발송됐습니다.'` (시뮬레이션 성공)
- **검증**: 토스트 텍스트

### O18 — 일괄 SMS 발송
- **조작**: `'미발송 X건 일괄 발송'` 버튼 클릭
- **기대**: Toast `'N개 리포트가 일괄 발송됐습니다.'`
- **검증**: 토스트 텍스트

### O19 — 일괄 발송 버튼 미발송 0건
- **조작**: 모든 리포트 발송 완료 후 일괄 발송 버튼 확인
- **기대**: 버튼 숨김 또는 Toast `'발송할 리포트가 없습니다.'`
- **검증**: 버튼 상태

### O20 — 이탈 위험 목록
- **조작**: 이탈 위험 페이지 접근
- **기대**: at_risk 수강생 목록 (비활성일수, 출석률, 위험점수 표시)
- **검증**: 테이블 또는 카드 rows 존재

## T. 교강사 (Teacher)

### T01 — 강의 업로드 폼 유효성 — 파일 없음

- **기대**: 버튼 비활성화 (disabled)
- **검증**: `button:has-text("업로드")` disabled

### T02 — 강의 업로드 폼 유효성 — 제목 없음
- **조작**: 파일 선택 + 과목 선택, 제목 비움 → 업로드
- **기대**: 버튼 비활성화 또는 에러
- **검증**: 제출 안 됨

### T03 — 강의 파일 업로드 및 SSE 진행상황
- **조작**: 제목 `'테스트 강의'` + 과목 `'국어'` + 샘플 오디오 파일 선택 → 업로드
- **기대**: 진행률 바 표시, 단계별 텍스트 변경 (`'음성 인식 중...'` → `'문제 생성 중...'`)
- **검증**: ProgressBar visible, 단계 텍스트
- **파일**: `/home/pabang/myapp/award/back/tests/fixtures/sample_lecture.mp3`

### T04 — 강의 업로드 완료
- **조작**: T03 완료 후 대기

## S. 수강생 (Student)

### S01 — 오늘의 문제 첫 번째 문제 표시
- **조작**: 문제 풀기 페이지 접근
- **기대**: 문제 카드, 선택지, `'1 / N'` 진행 표시
- **검증**: 문제 내용 텍스트 non-empty, 진행률 텍스트

### S02 — 선택지 선택 → 제출 버튼 활성화
- **조작**: 선택지 라디오 클릭
- **기대**: `'제출'` 버튼 활성화
- **검증**: 버튼 not disabled

### S03 — 선택지 미선택 → 제출 불가
- **조작**: 선택지 선택 안 하고 제출 버튼 클릭
- **기대**: 버튼 비활성화 또는 경고
- **검증**: disabled 상태

### S04 — 문제 제출 → 정답/오답 및 해설 표시
- **조작**: 선택지 선택 → `'제출'` 클릭
- **기대**: 정답 표시 (초록/빨강), 해설 텍스트 표시, `'다음 문제'` 버튼
- **검증**: 해설 텍스트 visible

### S05 — 다음 문제 이동
- **조작**: 제출 후 `'다음 문제'` 클릭
- **기대**: 다음 문제 카드 표시, 진행 `'2 / N'`
- **검증**: 진행 텍스트

### S06 — 이전 문제 버튼 — 첫 문제에서 비활성
- **조작**: 첫 문제에서 `'이전'` 버튼 확인
- **기대**: 버튼 없음 또는 비활성
- **검증**: 버튼 absent or disabled

### S07 — 이전 문제 이동
- **조작**: 두 번째 문제에서 `'이전'` 클릭
- **기대**: 첫 번째 문제로 이동
- **검증**: 진행 `'1 / N'`

### S08 — 마지막 문제 제출 → 결과 페이지
- **조작**: 모든 문제 제출 → `'결과 보기'` 클릭
- **기대**: `/student/quiz/result` 이동, 정답률 표시
- **검증**: URL, 점수 텍스트

### S09 — 오늘 문제 없을 때 빈 상태
- **조작**: 문제가 없는 상태에서 접근 (또는 초기화)
- **기대**: `'오늘의 문제가 없습니다.'` 메시지
- **검증**: 빈 상태 텍스트

### S10 — AI Q&A 새 대화 생성
- **조작**: Q&A 페이지 → `'새 대화'` 버튼 (`aria-label="새 대화 시작"`)
- **기대**: 빈 채팅창, 세션 목록에 새 항목
- **검증**: 채팅창 empty, 세션 추가

### S11 — AI Q&A 질문 전송 (Enter 키)
- **조작**: textarea에 질문 입력 → Enter 키
- **기대**: 메시지 전송, AI 응답 스트리밍 시작
- **검증**: user bubble 표시, AI 텍스트 점진적 표시

### S12 — AI Q&A Shift+Enter 줄바꿈
- **조작**: textarea에서 Shift+Enter 입력
- **기대**: 전송 안 됨, 줄바꿈만
- **검증**: textarea 개행 포함

### S13 — AI Q&A 스트리밍 중 입력 비활성
- **조작**: 메시지 전송 후 스트리밍 중 상태 확인
- **기대**: textarea disabled, send 버튼 disabled
- **검증**: disabled 상태

### S14 — AI Q&A 스트리밍 완료
- **조작**: 응답 스트리밍 완료 대기
- **기대**: AI bubble 완성, textarea 다시 활성화
- **검증**: textarea enabled, AI bubble non-empty

### S15 — AI Q&A 세션 전환
- **조작**: 사이드바에서 이전 세션 클릭
- **기대**: 해당 세션의 메시지 이력 표시
- **검증**: 메시지 목록 변경

### S16 — AI Q&A 에스컬레이션 발생 확인
- **조작**: 강의 관련 없는 질문 입력 (e.g., `'오늘 점심 뭐 먹었어?'`)
- **기대**: AI 응답 or 에스컬레이션 처리 (교강사에게 전달)
- **검증**: 응답 내용 기록 (버그 여부 판단)

### S17 — 모의고사 생성 시작
- **조작**: 모의고사 페이지 접근
- **기대**: `'AI가 맞춤형 모의고사를 생성 중입니다...'` 스피너
- **검증**: loading 텍스트 또는 spinner

- **조작**: 생성 완료 대기 (최대 30초)

### S19 — 모의고사 선택지 선택
- **조작**: 선택지 라디오 클릭
- **기대**: 해당 문제 네비게이션 점이 파란색으로 변경
- **검증**: 점 색상 클래스 변경

### S20 — 모의고사 문제 네비게이션
- **조작**: 문제 번호 점 클릭
- **기대**: 해당 번호 문제로 이동
- **검증**: 문제 번호 표시

### S21 — 모의고사 이전/다음 이동
- **조작**: `'다음'` → `'이전'` 순서로 클릭

- **기대**: D-day 숫자 표시 (수능까지 남은 일수)
